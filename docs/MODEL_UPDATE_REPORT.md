# 在反審查平台上治理留言濫用：機制、案例與教訓
## ——平台營運與用戶權益的雙重視角

**報告類型**：研究與決策分析報告（Research & Decision Brief）
**對應期間**：2026 年上半年 Matters 留言與文章濫用治理工程
**姊妹文件**：《Matters 平台留言濫用治理體系：2026 年度實施技術報告》（編年與 GitHub 變動對照）

## 摘要

本報告以分析之角度，檢視 Matters 平台 2026 年上半年所建立之濫用內容治理體系，並嘗試回答一個對抗審查平台而言具存在意義之問題：**一個以「保護表達」為宗旨的平台，如何在不背叛此宗旨的前提下，有效抑制垃圾濫用？**

我們主張，本期工作之價值不在任一單項技術，而在其所體現之一套**治理姿態**：(1) 以「群眾參與 + 可問責性」之守望相助機制，將審查權之行使置於稽核、透明與申訴之下；(2) 以「保護正當內容優先於攔截全部垃圾」之不對稱原則，校準自動化系統之決策邊界；(3) 以「測量誠實」作為一種治理紀律，反覆質疑量測指標，避免被「擋掉很多」之假象誤導。

報告之核心案例為文章偵測模型之偏差矯正：模型因訓練標籤之「作者連坐」而學會以高信度壓制華語政治評論、學術考據與個人創作；經規模化標籤矯正後，真實誤殺率由 24.0% 降至 1.6%，召回率維持約 92%。我們將此案例置於「演算法治理之失敗模式」之框架下分析，並自其引申出對平台營運與用戶權益之雙重啟示。

---

## 1. 緒論：反垃圾與反審查的內在張力

任何內容平台皆須處置濫用內容。但對 Matters 此類抗審查平台，濫用治理隱含一項在商業平台上不存在或不顯著之張力：

> **抑制垃圾與抑制異議，在技術上共用同一機制——一個分數加一個門檻；在價值上卻彼此對立。**

一個會「降低內容能見度」之自動系統，若判斷出錯，對一般商業平台僅是少推送了若干貼文；但對 Matters，誤判一篇政治評論而使其不再被看見，等同平台親手執行了它存在所要對抗之事——審查。此種誤判尤為隱蔽：它不刪除內容（故無顯著之刪除痕跡），僅令其沉默地失去能見度，且以「反垃圾」之名進行，無人察覺。

本報告所載之工程，即在此張力下展開。其根本立場可一語蔽之：**在不確定時，寧可漏看一篇垃圾，不願消音一篇表達。**

## 2. 群眾參與作為治理創新：守望相助模型

濫用治理之第一道防線並非機器，而是**社群**。守望相助（Community Watch）機制（建置編年見實施報告 §2）將部分內容審查權限以受控方式委派予經授權之社群成員，構成本體系之治理基石。其設計具三項值得分析之特徵：

### 2.1 權力之委派與其制衡

守望相助並非單純之「使用者檢舉」，而是賦予經授權成員直接移除留言之能力（`communityWatchRemoveComment`）。此種權力委派之風險顯而易見——濫權、誤判、針對性打壓。本體系以**建置順序本身**作為制衡之宣示：稽核表（PR #4763）、公開透明頁（#4764/#4765）、管理員覆核與申訴（#4771）皆**先於或同步於**處置權之開放而建立。換言之，「處置權之授予以可問責性為前提」並非事後補救，而是寫入了部署之時序。

### 2.2 透明與正當程序

- **稽核可見**：每一筆移除皆於 `Comment.communityWatchAction` 對外揭露公開 UUID、理由與時間，並提供來源連結（#4802）。
- **可還原、可申訴**：管理員覆核可還原留言、推翻處置，並以分離之 append-only 覆核事件（#4771/#4801）留存軌跡；還原與推翻皆通知當事人（#4791）。
- **讀寫一致性之堅持**：透明頁於還原後須立即反映新狀態，故稽核讀取改走主庫（#4775）——一個技術細節，卻是「透明承諾須即時兌現」之體現。
- **能見度之克制**：被移除留言若為文章唯一留言，仍以佔位形式保留於留言列（#4779），避免移除行為連帶抹除文章之對話脈絡。

### 2.3 隱私保全之群集

OSS 後台整合（#4822）引入 `contentHash`：使審查者得以將同一垃圾文本之跨次出現群集呈現、批次處置（`archiveUsers`），卻**不在後台重新散布該垃圾文本**。此為「營運效率」與「不放大有害內容」之折衷，是隱私工程思維滲入治理流程之一例。

> **小結**：守望相助之意義，在於它示範了「群眾參與」如何能在**不犧牲正當程序**之前提下擴大審查量能。機器（模型與 bot）之自動執法，正是建立於此一人本之、可問責之基礎上——bot 之 ring gate（實施報告 §5.2）即直接複用守望相助之「三人確認」證據作為自動打掃之依據，使自動化始終錨定於人類社群之既有判斷。

## 3. 機器學習層：從共用模型到 conformal abstention

### 3.1 專用化：承認留言與文章之異質

本期一項基礎性認識是：**留言垃圾與文章垃圾是不同之現象**。留言高度模板化（約 50 個模板家族），共用之文章模型於未見模板上召回僅 0.68；改採留言專用之 e5-small + 邏輯迴歸模型後升至 0.88（#4838）。此一專用化亦延伸至 bot 之掃描關鍵字——`45261d2` 之回退（將律師／翻譯／貸款等文章層 ring 關鍵字自留言 bot 移除）正是對「勿以一套規則套用異質現象」之自我修正。

### 3.2 Conformal abstention：將「機器的不確定」制度化

文章模型引入類別條件式 conformal 決策層（`spam-detection-serverless` PR #2）。其技術核心為：以每類校準分數陣列計算 p-value，保證 P(正當內容被封鎖 | 真 ham) ≤ ε，與類別先驗無關；輸出不再是二元之 spam/ham，而是 **allow / block / review** 三態，灰區交付人工，且每筆處置附可讀理由（對應歐盟 DSA 第 17 條之「處置理由聲明」精神）。

此設計之治理意涵深於其統計性質：它**將「機器的自信」與「機器的不確定」分離**，並把不確定之決定權交還給人。然而——此點至關重要——conformal 之 abstention 僅能挽救「分數位於灰區」之不確定案例；它**無法挽救「模型以高信度給出錯誤答案」之情形**。下一節之偏差案例，正是此一極限之具體呈現。我們亦誠實記錄：conformal 之「誤殺 ≤ ε」為期望／有限樣本性質，非硬保證（runbook，PR #3）。

## 4. 偏差案例研究：proxy-label 連坐與華語內容之壓制

### 4.1 症狀

於訓練截點後、模型未見之乾淨文章上驗收，舊文章模型以 **0.98–1.0 之高信度**將下列內容判為垃圾（分數 ≥ 0.94 門檻 → 排除曝光）：

- **政治評論**：〈國安部"反躺平"宣傳引起國人反感和反駁〉、〈躺平：基於改革開放…〉、〈習近平首試"硬碰硬"…〉
- **時事分析**：〈12月28日全球觀察：特朗普關稅…〉
- **學術考據**：〈談清儒偽造的"王年月偽器"《散氏盤》偽銘文〉
- **個人創作**：〈日記，超級生命密碼〉

此皆非邊緣案例，而正是平台最該保護之內容。量化（以逐篇確認之真實標籤）：**真實誤殺率 24%（@0.5）**。

### 4.2 根因：標籤層之「連坐」

模型之偏差源於訓練標籤。其垃圾正樣本很大一部分來自一個**代理標籤**——「作者被限制」（user_restriction）：只要一位作者因任何理由被限制，其名下**全部文章**即被標為垃圾，包括其合法之政治評論、學術文與日記。

模型遂習得一條危險之捷徑：**「華語政治／異議書寫」之特徵 ≈ 垃圾**。此為一種資料層面之連坐——一人受限，其全部表達被當作垃圾，再經模型將此偏見固化、規模化、自動化。

> 對一個反審查平台，這是最不可接受之失敗模式：**以「反垃圾」之名，行「壓制異議」之實，且為自動的、無人知曉的。** 此案例亦為「演算法治理」之一般性教訓提供了鮮明註腳：模型不會憑空學壞，它忠實地學會了我們（透過標籤）教給它的偏見；資料標註之政治，即是模型行為之政治。

### 4.3 模型學會的新邊界：結構，而非主題

矯正後之 v1 於同一批文章之表現：

| 文章 | 舊模型 | v1 | 判讀 |
|---|---|---|---|
| 國安部反躺平（政治評論） | 0.984 | **0.081** | 救回異議書寫 |
| 散氏盤偽銘文（學術考據） | 0.996 | **0.067** | 救回學術 |
| 超級生命密碼（個人日記） | 1.000 | **0.031** | 救回創作 |
| 新百勝賭場（博弈廣告） | 0.992 | 0.958 | 仍攔垃圾 |
| Call Girls（色情招攬） | 1.000 | 0.999 | 仍攔垃圾 |

**關鍵洞察：真正之垃圾不由「主題」定義，而由「結構」定義**——關鍵詞堆砌、導流至站外聯絡方式、為特定服務／品牌反覆置入、機械式跨帳號重複。一篇談「資產管理」之文章，可能是知識科普（ham），也可能是討債公司之形象洗白 SEO（spam）；差別不在它談什麼，而在它**怎麼寫、誰寫、要你做什麼**。v1 學會了看結構。

## 5. 矯正方法論

### 5.1 修標籤，而非「重新平衡」

先前曾試「rebalance 重訓」（調整類別比例），結果是將模型**洗鈍**：誤殺未降、召回反掉。此一失敗確立了一項認識：**偏差之根在標籤之錯誤，不在比例。** v1 直接矯正被連坐誤標之標籤（將約 2,649 篇合法內容翻回 ham 並加重學習），模型方真正學對。結果：

| | 真誤殺（@0.5） | 垃圾召回 |
|---|---|---|
| 舊模型 | 24.0% | 97% |
| **v1（修標籤）** | **1.6%** | 92% |

誤殺降 22 個百分點，召回僅掉 5——**保護了異議，未放縱垃圾。**

### 5.2 不對稱原則

判準明訂：將正當內容誤判為垃圾之代價，**遠大於**漏掉一篇垃圾；模糊一律傾向放行。此非工程上之任意選擇，而是 §1 所述價值之直接落實——少看到一篇廣告是小事，消音一篇政治評論是大事。

### 5.3 規模化標註：人在迴路、AI 規模化

矯正數萬筆被連坐之標籤，人工判不完，而平台不願將用戶內容送往外部 API。解法是以**平台同一信任邊界內之 AI 代理（Claude subagent）平行標註**：由主控訂判準與把關，多個代理分批逐篇判 ham/spam，產出矯正標籤，全程留下可審計之判準與理由。此一「人定原則、抽查、把關，AI 做規模化苦工」之模式，其意義超出本次個案，為標籤治理提供了一條兼顧規模與可問責之路徑。

成本面亦值得記錄：整個矯正、重訓、驗收、上線之 GPU 成本約 1 美元；**主要成本是判斷力與測量紀律，而非算力。**

## 6. 踩過的雷：測量誠實作為治理紀律

模型只是工作之一半；另一半是**不被自己的數字欺騙**。本期每一關鍵轉折，皆賴「先質疑數字、再下結論」而避免了錯誤決策。以下六例，共同指向一個治理層面之教訓。

1. **「100% 誤殺」之假象（量測管線 bug）**：首次驗收顯示 ham 被封鎖 97%。若當真，將誤判模型完全失效。追查發現係 payload 格式錯誤——以 JSON 字串而非原始文本送入端點。改正後方見真相。
   *教訓：異常到不合理之數字，先懷疑量測管線。*

2. **「乾淨 ham」其實不乾淨（代理污染）**：以「老帳號、未受限」作為「乾淨合法內容」之代理，算出誤殺 36%。然逐篇複核發現此代理約 37% 其實是垃圾（老帳號亦接外送茶、做貸款 SEO；一個 escort 帳號已 818 天、883 篇）。去污染後真誤殺方為 1.6%。
   *教訓：在被污染之標籤上算出之「誤殺率」嚴重失真；唯一可信之數字來自逐篇確認之乾淨樣本。*

3. **金融偽裝（領域知識之必要）**：數篇「防詐騙科普」「資產管理解析」一度被標為合法。細察其作者為貸款／討債帳號、結尾導流至自家產品——實為偽裝成知識之 SEO 垃圾。耐人尋味者：模型即便被（錯誤地）餵以這些當 ham，仍判其偏垃圾——**模型比標註者更早嗅到了垃圾之結構**。
   *教訓：偽裝型垃圾最難；判斷須看「發文帳號性質 + 結尾意圖」，不僅看文體。*

4. **分數與帳號年齡皆會騙人**：高分 ≠ 垃圾（誤判中文創作），老帳號 ≠ 良民（escort 老帳號）。能零誤殺分辨真垃圾者，是**複合訊號**（分數 ∧ 聯絡方式 ∧ 招攬詞），非任一單一特徵——此即留言三層告警 A 層之設計依據（#4851）。

5. **DB 約束漂移（實彈暴露）**：海巡 bot 之 Tier-1 檢舉於生產被 `report_reason_check` 約束拒絕（#4844）——GraphQL enum 與 DB 約束長期漂移而未被察覺，直到一個外部自動執法臂以真實流量觸發它。
   *教訓：自動化執法臂是最誠實之整合測試。*

6. **部署層之沉默陷阱**：IGW 公有子網無 egress 致 CodeBuild 逾時；SageMaker 輸出 gzip tar 而 Dockerfile 用 `tar -xf`；dispatch 之 boolean 預設值悄然覆寫版本庫變數使手動觸發回退 dry-run（coastguard `fd67b3f`）。
   *教訓：沉默之回退（silent fallback）比顯著之失敗更危險。*

> **這六雷之共同主題**：自動化內容治理最大之風險，不是模型不夠強，而是**評估時被自己的代理指標欺騙**，誤以為「擋掉很多」即是「做對了」。測量誠實，於是不只是工程美德，而是一種治理紀律——它是防止「演算法以正確之名行錯誤之實」的最後一道防線。

## 7. 討論

### 7.1 平台營運之立場

1. **垃圾照樣擋得住**：召回 92%，博弈／色情／詐騙／外送茶仍被攔；矯正偏差未犧牲防護。
2. **可持續改善之閉環**：處置結果（尤其人工推翻之誤殺）經 L1/L2/L3 留存體系回流為訓練資料，模型越用越準；且訓練訊號不因內容刪除而流失。
3. **低成本、可回滾、可重複**：約 1 美元重訓、一鍵部署、舊權重一鍵回滾、CI 全綠把關、curated 發布使垃圾防治得與其他功能解耦上線（#4849）。
4. **測量紀律之制度化**：將「先驗污染再下結論」「乾淨 held-out 才算數」寫入流程，避免未來被假指標誤導。
5. **群眾量能之槓桿**：守望相助使審查量能得以隨社群擴張，而 bot 之 ring gate 將社群之既有判斷自動化複用，形成人機協同之放大效應。

### 7.2 用戶權益之立場

1. **異議書寫重獲能見度**：被連坐壓制之政治評論、學術、創作，新內容即時恢復曝光；平台不再以反垃圾之名行審查之實。
2. **正當程序**：守望相助之稽核、透明頁、申訴、覆核，加上 conformal 之灰區送人工、處置帶理由（DSA 第 17 條精神），使機器與群眾之判斷皆非終局。
3. **能見度之克制**：處置上限為降低能見度而非刪除（「不刪除，只是不再被看見」）；門檻刻意保守（0.94），且留言層維持 notify-only、不啟用自動摺疊——寧可漏看垃圾，不願誤殺表達。
4. **透明與可問責**：每筆自動處置有可讀理由、可逆路徑、人工覆核層；外部執法臂（bot）受 reversed-rate 煞車約束，其移除一旦頻繁被推翻即自動停手。

### 7.3 一個未竟之平衡

本報告亦須誠實指出體系之限度：(1) 文章層之修復對**新內容**即時生效，但先前已被埋沒之合法舊文不予回溯重打分（本期決策）——歷史之誤殺未被追償。(2) 帳號層 ring 凍結仍屬候選階段，分級凍結與 shadow-first 尚未生產化。(3) 守望相助之權力委派，其長期之公平性仍須持續之稽核資料監測，本報告所載僅為機制之建立，非其長期效果之評估。

---

## 8. 結論

本期工作表面上是「若干模型與機制之上線」，本質上是一次**價值對齊**：讓自動內容治理系統——連同其背後之群眾參與機制——重新服膺平台「保護表達」之初衷，而非在無人察覺處侵蝕它。

對 Matters 此類平台，垃圾治理之每一道門檻、每一筆標籤、每一次權力委派，都是一個關於「什麼該被看見、由誰決定、如何問責」之微小決定。把這些決定做對、做得透明、把不確定交還給人——這不只是工程問題，是平台對用戶之承諾。本報告所欲傳達之核心，並非任一技術之優越，而是一種姿態：**在反審查平台上治理濫用，其成敗不在於攔截了多少垃圾，而在於是否在攔截垃圾的同時，始終守住了不誤傷表達的那條線。**
